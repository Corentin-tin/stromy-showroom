## Project Overview

This is a static website for Stromy company built with **Hugo** using the `hugo-saasify-theme`. Hugo is a static site generator that uses Go templates and markdown content to generate HTML. This theme is built with TailwindCSS for modern, responsive styling.

## Requirements

- Hugo Extended v0.80.0+
- Node.js v14.x+
- npm

## Development Commands

### Development Server
```sh
npm run start
```
- Runs TailwindCSS in watch mode
- Starts Hugo development server with live reload
- `-D` flag included (shows draft content)
- Default URL: http://localhost:1313

### Production Build
```sh
npm run build
```
- Builds and minifies TailwindCSS styles
- Generates static site files in `public/` directory with Hugo minification
- This is the command used by GitHub Actions for deployment

## Architecture & Content Management

### Content Structure

The hugo-saasify-theme uses a **modern content-driven architecture** with shortcodes and markdown:

- **`content/`**: All page content as markdown files
  - `content/_index.md`: Homepage content with shortcodes
  - `content/blog/`: Blog posts
  - `content/features/`: Feature pages
  - `content/docs/`: Documentation pages
  - Each page uses frontmatter for metadata and shortcodes for components

### Configuration-Based Settings

The theme relies on `hugo.toml` for global configuration:

- **Menu system**: Defined in `[[menu.main]]` sections
  - Supports dropdown menus via `has_submenu = true`
  - Submenu items defined in `[menu.main.params]` with `submenu` array
  - Footer menus: `menu.footer_column_1`, `menu.footer_column_2`, `menu.footer_column_3`

- **CTA Configuration**: Global call-to-action settings in `[params.cta]`
  - Customizable gradient colors and angles
  - Primary and secondary button configuration

- **Header/Footer**: Configured via `[params.header]` and `[params.footer]`
  - Logo, menu spacing, dropdown styling
  - Social media links in `[params.social]`
  - Sign in and Get Started button URLs

### Standard Hugo Directories

- **`content/`**: Markdown files for pages and blog posts
  - Use shortcodes for components (21 available shortcodes)
  - Blog posts use taxonomies: `categories` and `tags`
  - Frontmatter controls page metadata and features

- **`static/`**: Static assets (images, fonts, files)
  - Processed CSS goes to `static/css/style.css` (auto-generated by TailwindCSS)
  - Images: `static/images/`

- **`layouts/`**: Template overrides (only override theme templates when necessary)
  - `layouts/partials/custom-head.html` for custom JS/CSS includes

### Theme Customization Approach

**DO NOT edit files in `themes/hugo-saasify-theme/`**. Instead:
- Override layouts by creating same-path files in root `layouts/` directory
- Customize TailwindCSS: Edit `tailwind.config.js` for colors, fonts, and styling
- Configure via `hugo.toml` parameters
- Theme uses TailwindCSS utility classes (no predefined color schemes like old theme)

## Key Configuration Patterns

### Build Configuration (Required)
```toml
[build]
  writeStats = true

[build.buildStats]
  enable = true
```
These settings are required for TailwindCSS to work properly with Hugo.

### Theme-Specific Parameters
- `params.logo`: Path to logo image (e.g., "images/logo.svg")
- `params.description`: Site description for SEO
- `params.author`: Author name for metadata
- `params.googleAnalytics`: Google Analytics ID (production only)
- `params.googleTagManager`: Google Tag Manager ID (production only)

### Blog Configuration
- `params.blog.enable`: Enable/disable blog section
- `params.blog.cta.enable`: Show CTA in blog posts
- `params.blog.sidebar`: Configure sidebar widgets (recent, categories, tags, subscribe)

### Integration Services
- Google Analytics: `params.googleAnalytics = "G-XXXXX"`
- Google Tag Manager: `params.googleTagManager = "GTM-XXXXX"`
- Newsletter: Configure in `params.blog.sidebar.subscribe` (uses Formspree)

## Important Notes

- Theme is installed as git submodule at `themes/hugo-saasify-theme`
- Theme uses TailwindCSS 3.x for styling (no jQuery)
- 21 shortcodes available for rapid component development
- Multilingual support with i18n (configure in `hugo.toml` [languages] section)
- Performance optimized: 90+ Lighthouse scores
- Requires Hugo Extended (for SCSS/SASS processing)
